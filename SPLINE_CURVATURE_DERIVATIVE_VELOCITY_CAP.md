# Prepared geometric samples and initial jerk-derived velocity caps

Updated: 2026-07-17

This document describes the 65 feed-independent geometric samples generated by
the geometry producer and the related local velocity caps used by continuous
trajectory planning. It does not describe reachable-state timing, Ruckig phase
selection, correction passes, or final emitted-polynomial proof.

## Terminology

For a path parameterized by arc length `s`:

```text
q(s)    = axis-space path position [X,Y,Z,A,B,C]
q'(s)   = unit path tangent
q''(s)  = curvature vector
q'''(s) = arc-length derivative of the curvature vector
```

For a planar curve:

```text
q''' = kappa' N - kappa^2 T
```

The following quantities are distinct:

- Normal sharpness is `|kappa'|`, implemented as the magnitude of the component
  of `q'''` normal to the tangent.
- Full geometric jerk coefficient is `|q'''|`, including both the normal
  `kappa' N` term and tangential `-kappa^2 T` term.
- Coupled path jerk is the complete time-domain expression
  `q' j + 3 q'' v a + q''' v^3`.

The current Preview comb stores and displays both normal sharpness and the full
geometric jerk coefficient. Its primary magnitude is the full coefficient.

## Why samples are generated

Each `PreparedPathPiece` receives 65 samples at uniformly spaced path distances:

```text
distance[i] = piece_length * i / 64
i = 0 through 64
```

Both endpoints are included. The samples provide a stable geometry-only data
set for Preview diagnostics and can eventually be reused by trajectory timing.

Each `PreparedGeometricSample` stores:

- distance within the prepared piece;
- position;
- unit tangent;
- curvature vector;
- curvature-derivative vector `q'''`;
- normal sharpness;
- full geometric jerk coefficient.

These are geometric values. They do not depend on programmed feed, feed
override, acceleration limits, jerk limits, timing states, or servo period.

## Prepared spline representation

A prepared spline stores:

- degree;
- control points;
- knot vector;
- first-, second-, and third-derivative control nets;
- an immutable arc-length bracket table;
- a conservative maximum second parameter derivative used by chord-error work.

Derivative control nets are constructed once. Evaluating a sample does not
rebuild them.

The arc-length table supplies a bracket for inversion; it is not accepted as an
uncertified interpolation of distance to spline parameter. Exact requested
distances are checked using the adaptive integral and safeguarded Newton
iterations. Bit-exact requested distances may use the fixed inverse cache.

For the 65 ordered spline samples, `PreparedGeometry` also carries an ordered
parameter state from one sample to the next. This avoids restarting every
monotone inversion from unrelated parameter context.

## Parameter derivatives

Let a spline use ordinary parameter `u` and define:

```text
r1 = dr/du
r2 = d2r/du2
r3 = d3r/du3
w  = |r1|
T  = r1 / w
h  = dot(r1, r2)
```

The arc-length curvature vector is evaluated as:

```text
q'' = r2 / w^2 - r1 * dot(r1,r2) / w^4
```

The implementation computes the parameter derivative of that vector and then
divides by `w` to convert from differentiation by `u` to differentiation by arc
length. Written in the same form as the code:

```text
d(q'')/du =
      r3 / w^2
    - 3 r2 h / w^4
    - r1 * (dot(r2,r2) + dot(r1,r3)) / w^4
    + 4 r1 h^2 / w^6

q''' = (d(q'')/du) / w
```

This uses analytic spline derivatives. It is not a finite difference of nearby
sample positions.

## Arc and line samples

Lines have constant tangent and zero curvature and curvature derivative.

Arcs use the shared endpoint-exact arc reference. Position, tangent, curvature,
and curvature derivative are evaluated at path distance using that reference's
arc-length inversion policy. Rounded IJK arcs therefore reach both canonical
endpoints without pretending the accepted start and end radii are identical.

## Normal sharpness and full coefficient

At each sample, the producer projects `q'''` onto the tangent:

```text
tangential = dot(q', q''')
normal_part = q''' - q' * tangential

normal_sharpness               = |normal_part|
full_geometric_jerk_coefficient = |q'''|
```

For a planar circle, normal sharpness is zero because signed curvature is
constant, but the full coefficient is not zero:

```text
q''' = -kappa^2 T
|q'''| = kappa^2
```

This is why normal sharpness must not be used as a substitute for complete
geometric jerk.

## Initial velocity caps

If scalar path speed is `v`, scalar path acceleration is `a`, and scalar path
jerk is `j`, then axis-space derivatives are:

```text
axis velocity     = q' v
axis acceleration = q' a + q'' v^2
axis jerk         = q' j + 3 q'' v a + q''' v^3
```

The initial local sampling pass derives conservative candidates from individual
terms before the complete reachable-state solve.

### Programmed feed

The piece begins with its programmed-feed limit. Retained primitive sections
use their source entity feed. Junction blends use the arithmetic mean of their
two adjacent source feeds.

### Axis velocity

For each finite axis velocity limit:

```text
v <= axis_velocity_limit / |q'_axis|
```

### Centripetal acceleration

From the curvature term alone:

```text
v <= sqrt(path_acceleration_limit / |q''|)

v <= sqrt(axis_acceleration_limit / |q''_axis|)
```

### Geometric jerk

From the `q''' v^3` term alone:

```text
v <= cbrt(path_jerk_limit / |q'''|)

v <= cbrt(axis_jerk_limit / |q'''_axis|)
```

The selected initial velocity limit is the minimum applicable candidate across
programmed feed, aggregate limits, and all finite axis limits.

An experimental `curvatureDerivativeVelocityCapMultiplier` may scale the
jerk-derived candidate for offline investigation. Production correctness must
not rely on weakening the final proof.

## What the initial cap does not prove

The geometric-jerk cap isolates `q''' v^3`. Actual jerk also contains `q' j`
and `3 q'' v a`, which may reinforce or oppose it component by component.

Therefore the sampled cap is not the final feasibility decision. The planner
still performs:

- jerk-aware forward and backward reachability;
- bidirectional station optimization over complete `(v,a)` states;
- local Ruckig position solves;
- complete coupled acceleration and jerk checks at geometry boundaries;
- exact extrema checks over emitted axis-space cubic polynomials;
- local correction passes when an emitted constraint is violated.

## Current duplicate work

The geometry producer generates and stores these 65 samples for every prepared
piece. Preview consumes them directly.

The trajectory planner currently evaluates 65 locations again when building
its local limits, even when it received a `PreparedPathPiece` containing the
same sample locations. That duplicate evaluation remains unfinished
performance work.

The safe optimization is to pass the known prepared samples into the local
limit pass. Exact curve evaluation must remain available for:

- arbitrary timing stations;
- boundary PVA conversion;
- arc-length-certified inverse queries not already cached;
- adaptive ordered geometry verification;
- emitted-polynomial proof and correction.

Reusing the 65 samples should remove repeated derivative and inversion work; it
must not turn the sample grid into the authority for continuous geometry proof.
